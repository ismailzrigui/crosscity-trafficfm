# Contributing

This project is a scientific benchmark package. Contributions should improve reproducibility, data governance, validation, or manuscript clarity without expanding claims beyond executed evidence.

## Contribution Rules

- Do not add fabricated metrics, citations, datasets, DOIs, authors, or validation claims.
- Every new result must be generated by code or tied to a documented source.
- Any new dataset must include source URL, access date, license or terms note, period, variables, limitations, and citation guidance.
- Claims about safety, emissions, traffic control, or Morocco 2030 operations require matching evidence and should otherwise remain outside scope.
- Generated figures and tables should be reproducible from scripts.

## Suggested Development Flow

1. Update or add code in `scripts/`.
2. Re-run `scripts/run_pipeline.py`.
3. Re-run `scripts/run_tests.py`.
4. Rebuild the manuscript if article text or generated artifacts changed.
5. Update `data_manifest.md`, `source_registry.md`, and relevant reports when data or evidence changes.
